# Apex Cloud Consulting — Project Context Skill

This skill provides all agents with the necessary context about the company, target audiences, personas, and marketing strategy for building the Apex Cloud Consulting website.

---

## Company Overview

**Name:** Apex Cloud Consulting
**Industry:** IT Consulting — Cloud Solutions for the Automotive Industry
**Target Market:** Mid-sized automotive suppliers (50–500 employees)

### Vision
To make mid-sized automotive companies more efficient, transparent, and future-ready through cloud technologies and digital processes. The focus is on transforming paper-based and manual workflows into scalable, cloud-based systems that enable real-time data and automated processes.

### Mission
Supporting automotive mid-market companies in the cloud transformation of their business and production processes through:
- Analysis of outdated manual processes
- Development of digital, cloud-based target processes
- Implementation via cloud technologies (workflow automation, data collection)
- Integration of existing systems (ERP, production systems)
- Employee enablement

### Core Values
- **Practicality:** Solutions must work in day-to-day production environments
- **Transparency:** Clear processes, traceable results
- **Sustainability:** Long-term digitalization, not isolated solutions
- **Partnership:** Close collaboration with clients
- **Responsibility:** Secure handling of data and cloud technologies

### Services
- Cloud Migration
- Connected Car Infrastructure
- Data Analytics for OEMs
- Hybrid Cloud Architecture
- ERP System Integration
- Workflow Automation

## Personas

### Persona 1: Stefan Kruse (55) — "Der On-Premise-Protektor"
**Role:** IT Infrastructure Manager / Senior Sysadmin
**Location:** Stuttgart
**Education:** Vocational training as IT specialist (Fachinformatiker)

**Profile:**
Stefan has been with his company for over 25 years. He built the IT from scratch and sees cloud as a risk to data sovereignty. He prioritizes physical control, stability, and independence over modern trends.

**Pain Points:**
- Fear of losing control over data and systems
- Distrust of external providers
- Prefers one-time payments over subscription models
- Shadow IT from younger colleagues frustrates him

**Motivations:**
- Maximum system control and stability
- Independence from external vendors
- Retiring without security incidents

**Decision Style:**
Relies on proven industry standards and long-term colleagues' expertise. Seeks low-maintenance solutions that fit existing infrastructure. Prioritizes physical security over marketing trends.

**Preferred Channels:** LinkedIn (technical content only), IT trade magazines (c't), technical forums (Administrator.de), YouTube (hardware tutorials)

**How to reach Stefan:**
- Use technical depth, not marketing language
- Emphasize data sovereignty, GDPR compliance, hybrid solutions
- Provide detailed whitepapers and technical datasheets
- Address security concerns head-on with concrete answers
- Avoid buzzwords — speak his language

**Quote:** *"Früher wussten wir noch, was ein Bit ist. Heute werfen alle ihre Daten in eine Wolke und wundern sich über die Rechnung. Ohne mich."*

## Website Content Guidelines

### Tone of Voice
- **For Stefan:** Technical, precise, no buzzwords. Use data, specs, certifications. Build trust through expertise.
- **For Julia:** Results-oriented, concise, visual. Lead with outcomes. Make it easy to act.
- **Overall:** Professional, competent, partnership-focused. Not corporate-cold — warm and pragmatic.

### Must-Have Page Elements
1. **Hero Section:** Strong headline addressing the core pain (manual/analog processes in automotive), clear CTA
2. **Services:** Cloud Migration, Connected Car Infrastructure, Data Analytics for OEMs — each with concrete outcomes
3. **Why Apex:** Address Stefan's security concerns + Julia's ROI expectations simultaneously
4. **Case Studies / Social Proof:** Reference "Papierlose Produktion" type results
5. **Contact Form:** Minimal fields (Name, Company, Email, Message) — no friction for Julia
6. **Trust Signals:** DSGVO compliance badge, security certifications, partner logos
7. **Blog/Resources:** Whitepapers available for one-click download

### CTA Strategy
- Primary CTA: "Kostenloses Erstgespräch vereinbaren" (Julia-focused)
- Secondary CTA: "Whitepaper herunterladen" (Stefan-focused)
- Both CTAs should appear above the fold

## Technical SEO Requirements

- Meta title and description on every page targeting above keywords

- sitemap.xml and robots.txt
- Core Web Vitals: LCP < 2.5s, FID < 100ms, CLS < 0.1
- Lighthouse Score target: 90+ in all categories
- Mobile-first design (Julia uses mobile frequently)
